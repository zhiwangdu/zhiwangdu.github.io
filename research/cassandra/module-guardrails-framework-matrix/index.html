Operating Notes

  • 新增 guardrail 时至少要同步 Guardrails field、GuardrailsConfig getter、Config YAML field、GuardrailsOptions validation/getter/setter、GuardrailsMBean runtime method、GuardrailsConfigCommand reflected naming coverage、conf/cassandra*.yaml 注释模板、unit test 和本矩阵。
  • 对 query guardrail，fail 会抛 GuardrailViolatedException；对 background/null-state guardrail，fail 只通知日志、client warning ignore path、tracing ignore path 和 diagnostic event，不能中断 compaction/SSTable write。
  • Values 类型有 ignore action，最典型是 table properties；一致性级别使用 warned/disallowed，不使用 ignored。
  • nodetool setguardrailsconfig keyspaces_threshold <fail> <warn> 会在调用 MBean setter 前反转为 setter 的 <warn>, <fail> 顺序，文档和 runbook 必须明确这一点。
