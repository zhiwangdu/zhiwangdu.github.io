IP数据报格式

长度 = 固定部分20B+可变部分

固定部分

  • 版本：IPv4或者IPv6
  • 首部长度：单位是4B，最小为5
  • 区分服务：指期望获得那种类型的服务
  • 总长度：首部+数据的总长度，单位是1B
  • 标识、标志、片偏移：
  • 生存时间：TTL，IP数据报的保质期，每经过一个路由器就-1，变成0则丢弃
  • 协议：传输层传下来的报文使用的协议，TCP-6或UDP-17
  • 首部检验和：只检验首部
  • 源地址：32位
  • 目的地址：32位
  • 可选字段：长度可变，0～40B，用来支持排错，测量以及安全等措施
  • 填充：全0，把首部补成4B的整数倍。

数据报

无连接服务：不事先为分组的传输确定传输路径，每个分组独立确定传输路径，不同分组传输路径可能不同；

IP数据报格式

总长度单位是1B，片偏移单位是8B，首部长度是4B。

IP数据报分片

  • 标识：同一数据报的分片使用同一标识。

  • 标志：三位中只有两位有意义 X _ _

    中间位DF（Don’t Fragment），DF=1禁止分片，DF=0允许分片。

    最低位MF（More Fragment），MF=1后面还有分片，MF=0代表最后一片。

  • 片偏移：指出较长分组分片后，某片在原分组中的相对位置。以8B为单位。据此，除了最后一个分片，每个分片的长度一定是 8B的整数倍。

ICMP差错报告报文

  1. 终点不可达
  2. 源点抑制（基本不再使用）
  3. 时间超过
  4. 参数问题
  5. 改变路由（重定向）

不应发送ICMP差错报文的情况

  1. 对ICMP差错报告报文
  2. 对第一个分片的数据报片的所有后续数据报片
  3. 对具有组播地址的数据报
  4. 对具有特殊地址（如127.0.0.0或0.0.0.0）

路由算法

静态路由算法（非自适应路由算法）

管理员手工配置路由信息

优点：简便、可靠，在符合稳定、拓扑变化不大的网络中运行效果很好，广泛用于高度安全性的军事网络和较小的商业网络。

缺点：路由更新慢，不适用于大型网络。

动态路由算法（自适应路由算法）

路由器间彼此交换信息，按照路由算法优化出路由表项

优点：路由更新快，适用于大型网络，及时响应链路费用或网络拓扑变化。

缺点：算法复杂，增加网络负担。
