名词解释

  1. CSMA/CD
    Carrier Sense Multiple Access with Collision Detection
    载波监听多点接入/碰撞检测
    发送数据前，先侦听信道是否空闲，若空闲，则立即发送数据；若忙碌，则等待一段时间至信道中的信息传送结束之后在发送数据。若同时有两个或两个以上节点同时发送请求，则判定为冲突。若侦听到冲突，则立即停止发送数据，等待一段时间再重新尝试。先听后发，边听边发，冲突停发，随机延迟后重发。

  2. ARP
    Address Resolution Protocol
    地址解析协议,根据IP地址获取物理地址的一个TCP/IP协议

  3. ICMP
    Internet Control Message Protocol
    internet控制报文协议
    用于在IP主机、路由器之间传递控制消息。主要用于在主机与路由器之间传递控制信息，包括报告错误、交换受限控制和状态信息等等。

  4. OSPF
    Open Shortest Path First 开放式最短路径优先
    用于在单一自治系统内决策路由，通过路由器之间通告网络接口的状态来建立链路状态数据库，生成最短路径树，每个OSPF路由器使用这些最短路径构造路由表。

  5. BGP
    Border Gateway Protocol 边界网关协议
    允许一个AS与另一个AS进行通信。和其他的BGP系统交换网络可达信息。

  6. DNS
    Domain Name System 域名系统
    作为域名和IP地址相互映射的一个分布式数据库，能够使用户更方便的访问互联网，为不用去记住能够被机器直接读取的IP数串

  7. VPN
    Virtual Private Network 虚拟专用网咯
    在公用网络上建立专用网络，进行加密通讯。VPN网关通过对数据包的加密和数据包目的地址的转换实现远程访问。

  8. HTTP
    HyperText Transfer Protocol 超文本传输协议
    用于从www服务器传输超文本到本地浏览器传输协议。

  9. MPLS
    Multiple Protocol Label Switching 多协议标签交换
    在开放的通信网上利用标签引导数据告诉、高效传输的新技术。
    MPLS是利用标记进行数据转发的。当分组进入网络时，要为其分配固定长度的短的标记，并将标记与分组封装在一起，在整个转发过程中，交换节点仅根据标记进行转发。

  10. SCTP
    Stream Control Transmission Protocol，流控制传输协议
    SCTP是一个面向连接的流传输协议，它可以在两个端点之间提供稳定有序的数据传递服务。

  11. UDP
    User Datagram Protocol，用户数据报协议

    该协议是互联网协议族中传输层的协议，UDP只是在IP数据包服务上增加了分用、复用和差错检测功能。UDP是面向无连接的,发送数据不需要建立连接；使用尽最大努力交付，即不可靠交付；UDP是面向报文的，发送方UDP对应用程序交下来的报文既不合并也不拆分。

  12. 带宽

    在计算机网络中，带宽用来表示网络的通信线路传送数据的能力，因此带宽表示在单位时间内从网络中的某一节点到另一节点所通过的最高数据率，带宽的单位是“比特每秒”，b/s。

  13. 网络体系结构

    为了完成计算机之间的通信合作，把每个计算机互连的功能划分成定义明确的层次，规定了同层次进程通信协议及相邻层之间的接口及服务。将这些同层间通信的协议以及相邻层接口统称为网络体系结构。

  14. ADSL

    Asymmetric Digital Subscriber Line，非对称数字用户线

    是用数字技术对现有模拟电话用户线进行改造，使之能够承载宽带数字业务。ADSL的非对称是指下行带宽都远远大于上行带宽。

  15. VLAN

    Virtual LAN，虚拟局域网

    是由局域网段构成的与物理位置无关的逻辑组，而这些网段具有某些共同的需求，每一个VLAN的帧都有一个明确的标识符，指明发送这个帧的工作站是属于哪一个VLAN。

  16. CIDR

    Classless InterDomain Routing 无类别域间路由选择

    消除了A类，B类，C类地址以及划分子网的概念，同时CIDR把网络前缀都相同的连续IP地址组成一个“CIDR地址块”

  17. SMTP

    Simple Mail Transfer Protocol，简单邮件传送协议

    规定了在两个相互通信的SMTP进程之间应如何交换信息。SMTP采用客户——服务器方式，负责发送邮件的SMTP进程就是SMTP客户，负责接收邮件的进程就是SMTP服务器，至于邮件内部的格式，邮件如何储存以及邮件系统应如以多快的速度来发送邮件，SMTP都未作出规定。

  18. SNMP

    Simple Network Management Protocol，简单网络管理协议

    SNMP中管理程序和代理程序按客户——服务器方式工作。管理程序运行SNMP客户程序，而代理程序运行SNMP服务器程序。

  19. MAC

    Media Access Control，媒体访问控制

  20. RTT

    Round-Trip Time往返时间

  21. CRC

    Cyclic Redundancy Check 循环冗余检查

  22. VLAN

    Virtual Private Network 虚拟专用网

  23. 移动自组网络
    是一种无固定设施的无线局域网，其中的每一个移动站都要参与路由的发现和维护

  24. AS
    Autonomous System，自治系统

  5. 试简述RIP，OSPF和BGP路由选择协议的主要特点。

    RIP是一种分布式的基于距离向量的路由选择协议，是互联网的标准协议，其最大的优点就是简单。RIP协议的特点是：(1) 仅和相邻路由器交换信息。(2) 路由器交换的信息是当前本路由器所知道的全部信息，即自己的路由表。(3) 按笃定的时间间隔交换信息。

    OSPF最主要的特征就是使用分布式的链路状态协议。OSPF协议的特点是：(1)向本自治系统中所有的路由器发送信息。(2)发送的信息就是与本路由器相邻的所有路由器的链路状态(3)只有当链路状态发生变化时，路由器才向所有路由器用泛洪法发送此信息。

    BGP是不同自治系统的路由器之间交换路由器信息的协议，它采用路径响亮路由选择协议。BGP的主要特点是：(1)BGP在自治系统之间交换可达性信息。(2)自治系统之间的路由选择协议必须考虑有关策略(3)BGP只能是力求寻找一条能够到达目的网络且比较好的路由，而并非要寻找一条最佳路由。

  6. 试说明运输层在协议栈中的地位和作用。运输层的通信和网络层的通信有什么重要的区别？为什么运输层是必不可少的？

    (1)从通信和信息处理的角度看，运输层向它上面的应用层提供通信服务，它属于面向通信部分的最高层，同时也是用户功能中的最低层。当网络的边缘部分中的两个主机使用网络的核心功能进行端到端的通信是，只有主机的协议栈才有运输层，而网络核心部分中的路由器在转发分组时都只用到下三层的功能。

    (2)网络层提供点到点即主机与主机之间的逻辑通信，而运输层提供端到端应用进程之间的逻辑通信。

    (3)网络层只能将实现两个主机之间的通信，而真正需要通信的是主机中的应用进程，因此提供进程之间通信的传输层是必不可少的。

  7. 试举例说明有哪些应用进程宁愿采用不可靠的UDP，也不愿意采用可靠的TCP。
    (1)对于实时数据传输，如果宁可丢失少量分组也不要等待太晚到达的分组。

    (2)在连续的音频或视频流传输中，很少量的分组丢失对播放效果影响并不大。

    (3)当出现网络拥塞时，可能有的应用程序不愿放慢其报文段的发送速度

    (4)有的应用程序不需要TCP的可靠传输。

  8. 为什么说UDP是面向报文的，而TCP是面向字节流的？
    发送方的UDP对应用程序交下来的报文，在添加首部后就向下交付IP层。UDP对应用层交下来的报文，既不合并，也不拆分，而是保留这些报文的边界。在接收方的UDP，对IP层交上来的UDP用户数据报，在去除首部后就原封不动地交付上层的应用进程，。也就是说UDP一次交付一个完整的报文。

  11. 在TCP拥塞控制中，什么是慢开始、拥塞避免、快重传和快恢复算法？这里每一种算法各起什么作用？“乘法减小”和“加法增大”各用在什么情况下？

    慢开始：由小到大逐渐增大拥塞窗口数值。

    拥塞避免：当拥塞窗口大于慢开始门限ssthresh时，停止慢开始算法开始拥塞避免算法。拥塞避免算法是让拥塞窗口cwnd缓慢增大，即每经过一个往返时间RTT就把发送方的拥塞窗口加1。

    快重传：快重传算法要求接受方每收到一个失序的报文段后就立即发出重复确认,发送方接收到三个冗余确认后立即重传。

    快恢复：快重传之后进行乘法减小，门限值减半，后执行拥塞避免加法增大。

    乘法减小：只要发生拥塞，门限值就会减小到二分之一。

    加法增大：拥塞避免算法中。

  14. 5-46-使用具体例子说明为什么在运输层连接建立时要使用三报文握手。说明如果不这样做可能会出现什么情况。

    在上一个连接中，A向B发送的连接请求SYN报文滞留在网络中。于是A超时重传，与B建立了TCP连接，交换了数据，最后也释放了连接。

    但滞留在网络中的SYN报文段突然传送到了B。如果不使用三报文握手，B就会以为A现在请求建立TCP连接，于是就分配资源，等待A传送数据。但是A并没有要建立连接，也不会传送数据，于是B就白白等待A传送数据，浪费了资源。

    如果使用三报文握手，那么在B收到A发送的陈旧SYN报文后，就向A发送SYN报文段，A接收到此报文段后，根据ack=x+1就可以知道不应当理睬此报文段，因为A当前并没有发送seq=x的报文段。这时A返回复位报文段，拒绝建立连接。B收到后就知道不能建立TCP连接，就不会耗费资源等待A发送数据了。

  16. 流量控制和拥塞控制最主要的区别是什么？发送窗口的大小取决于流量控制还是拥塞控制？

    流量控制是在一条TCP连接中接受端用于限制发送端发送报文段速率的措施，流量控制只控制一个发送端;

    拥塞控制是用来控制TCP连接中发送端发送报文的速率，以免使互联网中的某处产生过载。拥塞控制可以控制多个发送端。

    当发送窗口小于拥塞窗口时，其取决于流量控制，即接受端的接收能力；当发送窗口大于拥塞窗口时，其取决于拥塞控制，即整个网络拥塞情况。

  1. 1-20.网络体系结构为什么要采用分层次的结构？
    网络体系结构采用分层次的结构，是因为分层可以把庞大而复杂的问题转化为若干小的局部问题，而这些较小的局部问题比较易于研究和处理。

  2. 1-24.试述五层协议的网络体系结构的要点，包括各层的主要功能。

  • 物理层——透明地传输比特流。
  • 数据链路层——将网络层的数据组装成帧，在两个相邻结点之间透明地传输帧中的数据。每一帧包括数据和必要的控制信息。
  • 网络层——负责为分组交换网上的不同主机提供通信服务。
  • 传输层——负责向两个主机中的进程提供通信服务。
  • 应用层——直接为用户的应用进程提供服务。
  1. 1-26.试解释一下名词：协议栈、实体、对等层、协议数据单元、服务访问节点、客户、服务器、客户-服务器方式。
  • 协议栈：由于计算机网络的体系结构采用了分层结构，因此不论在主机还是路由器中的协议都有好几层。这一层一层的协议很像堆栈的结构，所以就把这些协议层称为协议栈。
  • 实体：表示任何可发送或接收信息的硬件或软件进程。在许多情况下，实体就是一个特定的软件模块。
  • 对等层：在网络体系结构中，通信双方实现了同样功能的层。
  • 协议数据单元：通常记为PDU，它是对等实体之间进行信息交换的数据单元。
  • 服务访问节点：通常记为SAP，在同一系统中相邻两层的实体进行交互的地方，通常称为服务访问节点。
  • 客户：在计算机网络中进行通信的应用进程的服务请求方。
  • 服务器：在计算机网络中进行通信的应用进程的服务提供方。
  • 客户-服务器方式：C/S

    物理层考虑的是怎样才能在连接各种计算机的传输媒体上传输数据比特流，它的任务就是透明地传送比特流。

  2. 2-04.试解释一下名词：数据，信号，模拟数据，模拟信号，基带信号，带通信号，数字数据，数字信号，码元，单工通信，半双工通信，全双工通信，串行传输，并行传输。

  • 数据：是运送消息的实体。
  • 信号：是数据的电气或电磁的表现。
  • 模拟数据：即连续数据，即数据的变化是连续的，如人说话的声音数据。
  • 模拟信号：即连续信号，其特点是代表消息的参数的取值是连续的。入门打电话时，声波通过话筒变成了连续变化的电信号。
  • 基带信号：即来自信源的信号，也就是基本频带信号。如计算机输出的代表各种文字或图像文件的数据信号。
  • 带通信号：把基带信号的频率范围搬移到较高的频段以便在信道中传输。
  • 数字数据：离散数据，即数据的变化是不连续的。
  • 数字信号：离散信号，其特点是代表消息的参数的取值是离散的。
  • 码元：码是信号元素和字符之间的事先约定好的转换。码元是码所包含的元素，在简单的二进制编码中，一个码元就是一个比特；在比较复杂的编码中，一个码元可一包含多个比特。
  • 单工通信：单向通信，只能有一个方向的通信而没有反方向的交互。
  • 半双工通信：双向交替通信，通信的双方都可以发送消息，但不能同时发送。
  • 全双工通信：双向同时通信，通信的双方可以同时发送或接收消息。
  • 串行传输：数据在传输时是逐个比特按照时间顺序传输的。
  • 并行传输：数据在传输时采用了n个并行的信道。在每一条信道上，数据仍然是串行传输。

    许多用户通过复用技术就可以共同使用一个共享信道来进行通信。

    常用的复用技术有：频分复用，时分复用，波分复用和码分复用。

  3. 2-14.试写出下列英文缩写的全文，并进行简单解释。

  • FDM：Frequency Division Multiplexing，频分复用。给每个信号分配唯一的载波频率并通过单一媒体来传输多个独立信号的方法。
  • TDM：Time Division Multiplexing，时分复用。把多个信号复用到单个硬件传输信道。
  • STDM：Statistic TDM，统计时分复用。又称异步时分复用，是一种改进的时分复用，它能明显地提高信道的利用率。
  • WDM：Wavelength Division Multiplexing，波分复用。就是光的频分复用。人们借用载波电话的频分复用的概念，就能做到使用一根光纤来同时传输多个频率很接近的光载波信号。
  • DWDM：Dense WDM，密集波分复用。是波分复用的一种表现形式。

  1. 3-01.数据链路与链路有何区别？电路接通了与数据链路接通了的区别何在？
    链路就是从一个节点到相邻结点的一段物理线路，中间没有任何交换节点；数据链路是在链路的基础上增加了必要的通信协议。

  2. 3-02.数据链路层中的链路控制包括哪些功能？试讨论把数据链路层做成可靠的链路层有哪些优点和缺点。
    数据链路层的功能：封装成帧、透明传输和差错检测。

    可靠链路的优点是可以是网络中的某个节点及早发现传输中出了差错，因而可以通过数据链路层的重传来纠正这个差错。

    缺点：又是高层协议使用的是不可靠传输UDP。UDP并不需要求重传有差错的数据。此时可靠的链路层就会牺牲了UDP的实时性。

  3. 3-14.常用的局域网的网络拓扑有哪些种类？现在最流行的是那种结构？为什么早期的以太网选择选择总线拓扑结构而不是用星型拓扑结构，而现在却改用星型拓扑结构？
    常用的局域网的网络拓扑有星型网、环形网和总线网。

    现在最流行的是星型网。

    在局域网发展的早期，人们都认为有源器件比较容易出故障，所以认为无源的总线网会更加可靠，而星型拓扑结构的中心使用了有源器件。但是实践证明，连接有大量站点的总线式以太网，由于接插件的插口较多，反而很容易出现故障，现在专用的ASIC芯片的使用可以吧星型结构的集线器做得非常牢靠。因此现在的以太网一般都使用星型结构的拓扑。

  4. 3-15.什么叫做传统以太网？以太网有哪两个标准？
    传统以太网就是最早流行的10Mbit/s速率的以太网。

    主要有的两个标准：DIX Ethernet V2和IEEE 802.3

  5. 3-19.以太网使用的CSMA/CD协议是以争用方式接入到共享信道的。这与传统的时分复用TDM相比优缺点如何？
    网络上负荷较轻时，CSMA/CD协议很灵活，哪个站向发送就可以发送，而且发生碰撞的概率很小，但是此时使用TDM效率就很低；但是当网络负荷较大时，CSMA/CD引起的碰撞很多，重传经常发生，因而效率大大降低，而此时TDM效率很高。
