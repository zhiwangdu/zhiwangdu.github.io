# 类加载

-   在Java代码中，类型的加载、连接和初始化过程都是在程序运行期间完成的。

## 类加载器

Java虚拟机结束生命周期的情况

1.  执行了System.exit()方法
2.  程序正常执行结束
3.  程序在执行过程中遇到了异常或者错误
4.  操作系统出现错误

## 类的加载、连接和初始化概述

-   加载：查找并加载类的二进制数据，
-   连接：三个阶段  
    

1.验证：确保被加载的类的正确性  
  
2.准备：为类的静态变量分配内存，并将其初始化为默认值  
  
3.解析：把类中的符号引用转换为直接引用  

-   初始化：为类的静态变量赋予正确的初始值

### 例子

```
public class Test{
    //在准备阶段会为a分配内存，并赋值为默认值0，
    //在初始化阶段才会将1赋值给a
    public static int a = 1;
}
```

## 类的加载

-   加载：把二进制文件加载到内存里，将其放在运行时数据区的方法区内，然后在内存中创建一个java.lang.Class对象用来封装类在方法区内的数据结构。
    
    ### 加载.class文件的方式
    
-   从本地系统直接加载
-   通过网络下载class文件
-   从zip,jar等归档文件中加载.class文件
-   从专有数据库中提取
-   将java源文件动态编译为.class文件

## 类的使用

-   Java程序对类的使用可分为两种，主动使用和被动使用
-   所有Java虚拟机实现必须在每个类或接口被Java程序 *首次主动使用* 时才初始化他们

### 主动使用（七种）

-   除了这七种情况，其他使用Java类的方法都被看作是对类的 *被动使用* ，都不会导致类的初始化
-   常量在编译阶段会存入到调用这个方法的类的常量池中，本质上，调用类并没有直接引用到定义常量的类，所以不会触发定义常量的类的初始化。

1.  创建类的实例
2.  访问某个类的接口的静态变量，或者对该静态变量赋值
3.  调用类的静态方法
4.  反射
5.  初始化一个类的子类
6.  Java虚拟机启动时被标明为启动类的类
7.  根据java.lang.invoke.MethodHandle实例的解析结果进行初始化

### 例如

```
public class Test1 {

    public static void main(String[] args) {
        /**
         * 输出结果
         * this is parent
         * this is children
         * child,s str
         * 
         * parent和child都初始化了
         */
        System.out.println(Children.str2);

        /**
         * 输出结果
         * this is parent
         * parent's str
         * 
         * 初始化了parent而没有初始化child
         */
        System.out.println(Children.str);


        /**
         * 输出结果
         * MM's str
         *
         * final是常量，不会初始化MM类
         */
        System.out.println(MM.s);
    }
}

class Parent{

    public static String str = "parent's str";

    static {
        System.out.println("this is parent");
    }
}

class Children extends Parent{

    public static String str2 = "child,s str";

    static {
        System.out.println("this is children");
    }
}

class MM {
    //引用s字符串时不会触发MM类的初始化
    public static final String s = "MM's str";

    static {
        System.out.println("this is MM");
    }
}
```

### 类的加载、主动使用过程

```
package classloader;

public class Test6 {

    public static void main(String[] args) {
        P p = P.getInstance();

        System.out.println("counter1: " + p.counter1);
        System.out.println("counter2: " + p.counter2);
        /**
         * 输出：
         * counter1: 1
         * counter2: 0
         *
         * 在准备阶段为静态变量分配内存并赋值为默认值，即counter1=0，counter2=0，p=null
         * main方法使用了类P中的静态方法 getInstance()，即对类的主动使用，对类进行初始化
         * 初始化的过程从上到下进行赋值
         */
    }
}

class P {
    public static int counter1;

    public static P p = new P();

    private P() {
        counter1++;
        counter2++;
    }

    public static int counter2 = 0;

    public static P getInstance() {
        return p;
    }
}
```
