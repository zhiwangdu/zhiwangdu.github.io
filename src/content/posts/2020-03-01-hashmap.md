# Java 7 和 Java 8的区别

Java7的HashMap结构为数组+链表，Java8中为数组+链表/红黑树。

## 加入红黑树的原因

当hash碰撞过多时，链表长度会过长，哈希表的查询效率会退化为O(n),此时将链表改为红黑树结构可以提高效率至O(logn)。

# HashMap中常量的意义

-   `static final int DEFAULT_INITIAL_CAPACITY = 1 << 4;`  
    默认数组的长度为16，且数组长度必须为2的幂。  
    原因：计算数组下标的方式为：(length-1) & hash,只有2的幂减一和hash值做与运算才能算出下标。若不是2的幂，length-1后和hash做与运算有某些位置永远都为0，导致某些位置永远都是空值，增加了哈希碰撞的几率。
-   `static final float DEFAULT_LOAD_FACTOR = 0.75f;`  
    负载因子，是判断是否需要扩容的条件，当容量大于当前容量和负载因子的乘积时进行扩容。
-   `static final int TREEIFY_THRESHOLD = 8;`  
    链表是否需要变为红黑树的判断条件，每个位置上元素的个数满足泊松分布，个数大于8的概率非常小，所以当位置上的元素大于8时才变为树结构。

# put的过程

1.  计算key的hash值(key的高16位和key的哈希值做异或运算，增加随机性，减少哈希碰撞)
2.  根据hash值计算出下标，判断是否发生碰撞，若没有碰撞，则直接加入。
3.  如果散列表为空，则调用resize()方法初始化散列表。
4.  若发生碰撞，则判断此处为链表还是树，再进行插入。在向链表中插入时，若链表长度大于变为红黑树的阙值8，若大于，则转化为红黑树。
5.  加入完成后，判断散列表是否需要扩容。

# get的过程

根据key的hash值计算出下标，然后遍历数组下标处的链表或者红黑树，取出key对应的值。空间复杂度为O(1);

# HashMap线程不安全

扩容时可能会出现环形链表，get的时候就会死循环，产生死锁。
